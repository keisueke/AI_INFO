# 文章作成ガイドライン

このドキュメントは、AI_INFOサイトのコンテンツ作成時に参照すべき文章作成のガイドラインです。

## 基本原則

### 読みやすさを最優先に
- 専門用語は必要最小限に
- 難しい概念は具体例で説明
- 文章は簡潔に、余計な装飾は避ける

## 見出しの使い方

### 階層構造

```
h1: ページタイトル（1ページに1つのみ）
  └─ h2: 主要セクション（目次に表示される）
      └─ h3: サブセクション（目次に表示される）
          └─ h4: さらに細かい区分（必要最小限、目次には表示しない）
```

### 見出しのルール

1. **h1（ページタイトル）**
   - 1ページに1つのみ
   - ページの内容を端的に表現
   - 例: 「GitHub Copilotの特徴」「画像生成AIのTips」

2. **h2（主要セクション）**
   - ページ全体の構成を決める見出し
   - 3-7個程度が読みやすい
   - 目次に必ず表示される
   - 例: 「Copilotとは」「効果的な使い方」「制約事項と注意点」

3. **h3（サブセクション）**
   - h2内の詳細な説明
   - 目次に表示される（ネストして表示）
   - 例: 「1. コンテキストを理解した提案」「2. 複数行のコード生成」

4. **h4（細かい区分）**
   - 必要最小限の使用
   - 目次には表示されない
   - 通常はカードやボックス内の見出しとして使用

### 見出しの書き方

- **具体的な内容を示す**: 「概要」「詳細」ではなく「Copilotとは」「実践方法」など
- **質問形式も可**: 「なぜ重要か？」「どのように使うか？」など
- **統一感を保つ**: 同じレベルの見出しは同じスタイルで

## 段落と本文の書き方

### 段落の長さ

- **1段落は3-5文程度を目安**
- 長い段落（7文以上）は分割する
- 1文が長すぎる場合は、2-3文に分割

### 段落の構造

1. **導入文**: その段落で説明する内容を端的に示す
2. **説明文**: 詳細な説明や具体例
3. **まとめ文（必要に応じて）**: 要点の再確認

### 本文の書き方

- **主語・述語を明確に**: 「〜です」「〜します」を明確に
- **受動態を避ける**: 「〜されます」より「〜します」を優先
- **箇条書きを活用**: 複数の項目がある場合はリストを使用
- **具体例を入れる**: 抽象的な説明には必ず具体例を添える

### 良い段落の例

```
✅ 良い例:
GitHub Copilotは、コードエディタ内でリアルタイムにコード補完を提供します。
Visual Studio Code、JetBrains IDE、Neovimなど、主要なエディタで利用できます。
インライン補完とチャット機能により、効率的な開発が可能になります。
```

```
❌ 悪い例（長すぎる）:
GitHub Copilotは、OpenAIのGPTモデルをベースにしたAIアシスタントで、
コードエディタ内でリアルタイムにコード補完を提供し、Visual Studio Code、
JetBrains IDE、Neovimなど、主要なエディタで利用でき、インライン補完と
チャット機能により、効率的な開発が可能になり、関数名、変数名、コメントなどから、
あなたが求めているコードを推測して提案してくれます。
```

## 目次の配置

### 配置位置

- **ページの冒頭、h1の直後に配置**
- ページの内容を把握できる最初の場所

### 目次に含める内容

- h2（主要セクション）: 必ず含める
- h3（サブセクション）: 主要なものは含める
- h4以下: 含めない

### 目次の効果

- 読者が全体像を把握できる
- 必要な箇所に直接ジャンプできる
- 長いページでも迷わない

## コードブロックの使い方

### 使いどころ

- **実装例**: 具体的なコードを示す場合
- **設定例**: 設定ファイルやコマンドの例
- **比較**: 良い例と悪い例の対比

### コードブロックの書き方

```typescript
// コメントで意図を説明
// この関数はユーザー情報を取得する
async function fetchUser(id: string) {
  // 実装内容
}
```

- **言語指定**: TypeScript、JavaScript、JSONなど適切な言語を指定
- **コメント**: コードにコメントを入れて説明を補足
- **簡潔に**: 必要最小限のコードを示す

## スタイリングコンポーネントの使い方

### CopilotTip

**使いどころ**: 重要なヒントやベストプラクティス

```tsx
<CopilotTip>
  コメントで意図を明確に説明すると、Copilotの提案精度が向上します。
</CopilotTip>
```

- 短く、要点を絞る
- 1-2文程度が理想

### TryItOut

**使いどころ**: 実践的な練習やステップバイステップの手順

```tsx
<TryItOut>
  以下の手順で試してみましょう：
  <ol>
    <li>ステップ1</li>
    <li>ステップ2</li>
  </ol>
</TryItOut>
```

- 具体的な手順を示す
- 番号付きリストや箇条書きを活用

### CodeBlock

**使いどころ**: コード例、コマンド、設定など

- 言語指定を忘れない
- コメントで説明を補足

## リストの使い方

### 箇条書きリスト（ul）

- 順序が重要でない項目
- 複数の選択肢や例
- 特徴やメリット・デメリット

### 番号付きリスト（ol）

- 順序が重要な手順
- 段階的なプロセス
- ステップバイステップの説明

### リストの書き方

- **各項目は簡潔に**: 1項目1-2文程度
- **並列構造**: 同じレベルで同じ形式
- **空行で区切る**: リストの前後には空行を入れる

## ページ構造のテンプレート

```tsx
<h1>ページタイトル</h1>

<p>ページの概要を2-3文で説明</p>

{/* 目次 */}
<TableOfContents />

<h2>主要セクション1</h2>

<p>セクションの説明</p>

<h3>サブセクション1</h3>
<p>詳細な説明</p>

<h3>サブセクション2</h3>
<p>詳細な説明</p>

<h2>主要セクション2</h2>
...
```

## チェックリスト

コンテンツ作成時は以下を確認：

- [ ] h1は1つだけか
- [ ] h2は3-7個程度か
- [ ] 段落は3-5文程度か
- [ ] 長い段落は分割されているか
- [ ] 目次が追加されているか（長いページの場合）
- [ ] 具体例が含まれているか
- [ ] コードブロックにコメントがあるか
- [ ] リストが適切に使われているか
- [ ] 専門用語に説明があるか
